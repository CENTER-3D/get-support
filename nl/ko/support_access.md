# 지원 케이스 작업을 위한 사용자 액세스 지정
{: #access}

기본적으로 계정의 사용자에게 케이스를 작성하거나 업데이트하거나 검색하거나 보는 데 필요한 액세스가 없습니다. 계정 소유자는 Identity and Access Management(IAM) 액세스 정책을 지정하여 사용자 액세스를 제공해야 합니다. 지원 센터 계정 관리 서비스를 사용하여 지원 케이스에 대한 작업을 수행하기 위한 사용자 액세스를 지정하십시오.
{:shortdesc}

지원 센터 액세스 정책은 지원 케이스 작업을 위한 사용자 액세스를 제공합니다. 하지만 해당 액세스 정책은 사용자가 계정의 다른 사용자를 보기 위한 액세스를 제공하지 않습니다. 따라서, 계정 소유자가 [**사용자 목록 가시성 설정**](/docs/iam/userlist.html#userlistview)을 **제한된 보기**로 설정하는 경우, 케이스를 관리하는 액세스를 가진 사용자는 다른 사용자 또는 계정 소유자에게 지정된 케이스에 액세스할 수 없습니다. 사용자가 모든 케이스를 볼 수 있도록 하려면 뷰어 역할이 지정된 사용자 관리 계정 관리 서비스에 대한 IAM 액세스 정책을 지정하여 추가 액세스를 제공해야 합니다. 

IAM 액세스 정책을 사용하여 모든 케이스에 대한 액세스 대신 계정의 특정 케이스에 대한 전체 액세스를 특정 사용자에게 제공하려면, 케이스를 작성할 때 사용자 이메일을 입력하여 해당 사용자를 케이스에 추가하십시오. 작성된 케이스에 사용자를 추가하여 계정의 해당 케이스만 보고 편집하고 업데이트하는 액세스를 갖습니다. 또한 케이스에 대한 업데이트가 있을 때 알림을 수신합니다.

이전에 클래식 인프라 권한을 사용하여 지원 케이스 액세스를 지정한 클래식 인프라 사용자의 경우 이제 [마이그레이션된 클래식 인프라 권한 액세스 그룹](/docs/iam/infrastructureaccess.html#predefined)에서 해당 권한을 사용할 수 있습니다. 마이그레이션된 권한 액세스 그룹에는 뷰어 역할이 지정된 사용자 관리 서비스에 대한 IAM 정책이 포함됩니다.

계정 관리 서비스(예: 지원 센터 서비스 또는 사용자 관리 서비스)에 대한 IAM 액세스 정책을 사용자에게 지정하려면 다음 단계를 완료하십시오.

1. 메뉴 표시줄에서 **관리** &gt; **액세스(IAM)**를 클릭한 다음 **사용자**를 선택하십시오.
2. 목록에서 사용자를 선택하십시오.
3. **액세스 정책**을 클릭하십시오.
4. **액세스 지정**을 클릭하십시오.
5. **계정 관리 서비스에 대한 액세스 지정**을 선택하십시오.
6. 목록에서 서비스를 선택하십시오. 
5. 의도한 액세스를 지정할 역할을 선택하십시오.

다음 표를 확인하여 각 역할에 포함된 권한을 정확하게 파악하십시오.

|역할 |조치 | 
|--------|---------------|
|뷰어  | 케이스 보기 및 검색 |
|관리자 | 케이스 보기, 검색, 작성 및 업데이트 |
